# 16BitHero - Game Design Document (Overview)

## 📋 문서 정보
- **프로젝트명**: 16BitHero
- **장르**: 2D 탑다운 웨이브 서바이벌 액션 로그라이크
- **참고 게임**: Brotato
- **아트 스타일**: 16비트 레트로 픽셀 아트
- **작성일**: 2025-12-04
- **버전**: 0.1

---

## 🎮 게임 개요

### 핵심 컨셉
플레이어는 16비트 스타일의 영웅이 되어 끊임없이 몰려오는 적들의 웨이브를 견뎌내며 생존하는 게임입니다. 브로타토의 핵심 메커니즘을 기반으로, 자동 공격 시스템과 스탯 기반 성장을 통해 점점 강해지는 캐릭터를 육성합니다.

### 핵심 게임 루프
```
웨이브 시작 → 적 처치 & 자원 수집 → 웨이브 종료 → 상점/업그레이드 → 다음 웨이브
```

### 주요 특징
- **자동 공격**: 무기가 자동으로 가장 가까운 적을 조준하고 공격
- **다양한 무기 시스템**: 근접/원거리 무기를 최대 6개까지 장착 가능
- **스탯 기반 성장**: 13가지 스탯을 통한 캐릭터 커스터마이징
- **웨이브 시스템**: 시간 제한이 있는 웨이브를 견디며 생존
- **아이템 드랍**: 경험치, 골드, 재료 등을 수집하여 성장

## 🔧 현재 구현 상태

### ✅ 구현 완료
1. **캐릭터 시스템**
   - 플레이어 이동 (조이스틱 기반)
   - 체력 시스템
   - 회피(Dodge) 시스템
   - 레벨 시스템 (기본 구조)
   - 아이템 자동 수집 (범위 내)

2. **무기 시스템**
   - 무기 베이스 클래스 구조
   - 자동 조준 (Auto-Aim)
   - 근접 무기 (MeleeWeapon)
   - 원거리 무기 (RangeWeapon)
   - 투사체 시스템 (Bullet)
   - 크리티컬 시스템
   - 넉백 시스템

3. **적 시스템**
   - 적 베이스 클래스
   - 적 AI (플레이어 추적)
   - 적 체력/데미지 시스템

4. **웨이브 시스템**
   - 웨이브 관리자
   - 스폰 시스템 (시간 기반, 확률 기반)
   - 스폰 인디케이터 (적 출현 예고)
   - 웨이브 타이머 UI
   - 아레나 경계 체크

5. **스탯 시스템**
   - 13가지 플레이어 스탯 정의
   - 스탯 계산 시스템 (기본 스탯 + 추가 스탯 + 배율)
   - 스탯 UI 표시

6. **드랍 시스템**
   - 드랍 아이템 베이스 클래스
   - 자동 수집 메커니즘

7. **UI 시스템**
   - 스탯 UI
   - 웨이브 정보 UI
   - 일시정지/재개 기능

8. **기타**
   - 카메라 컨트롤러
   - 게임 매니저 (싱글톤)
   - ScriptableObject 기반 데이터 관리

### 🚧 부분 구현 / 주석 처리됨

   - 오디오 매니저 미구현 (무기 공격 사운드 주석 처리)

4. **플레이어 스탯 연동**
   - 무기에 플레이어 스탯 적용 로직 주석 처리

## 🎲 핵심 시스템 개요

### 1. 플레이어 스탯 시스템
플레이어의 능력치를 결정하는 13가지 스탯:

| 스탯 | 설명 | 현재 구현 |
|------|------|-----------|
| **Damage** | 기본 공격력 | ✅ |
| **MeleeAttack** | 근접 공격력 보너스 | ✅ |
| **RangeAttack** | 원거리 공격력 보너스 | ✅ |
| **AttackSpeed** | 공격 속도 | ✅ |
| **CriticalChance** | 치명타 확률 | ✅ |
| **CriticalMultiplier** | 치명타 배율 | ✅ |
| **MoveSpeed** | 이동 속도 | ✅ |
| **MaxHealth** | 최대 체력 | ✅ |
| **Range** | 공격 범위 | ✅ |
| **HealthRecoverySpeed** | 체력 회복 속도 | ⚠️ 정의만 |
| **Armor** | 방어력 | ⚠️ 정의만 |
| **Luck** | 행운 (드랍률 등) | ⚠️ 정의만 |
| **Dodge** | 회피율 | ✅ |
| **LifeSteal** | 생명력 흡수 | ⚠️ 정의만 |

**스탯 계산 공식**:
```
최종 스탯 = ((기본 스탯 + 추가 스탯) × 스탯 배율) × 0.01
```

### 2. 무기 시스템
- **무기 타입**: 근접(Melee), 원거리(Range)
- **장착 제한**: 최대 6개 (브로타토 방식)
- **자동 공격**: 범위 내 가장 가까운 적을 자동 조준
- **무기 속성**:
  - 공격력 (Damage)
  - 공격 속도 (Attack Delay)
  - 사거리 (Range)
  - 넉백 (Knockback)
  - 크리티컬 확률/배율

### 3. 웨이브 시스템
- **웨이브 구조**: 각 웨이브는 여러 스폰 패턴으로 구성
- **스폰 설정**:
  - 스폰 시작 시간
  - 스폰 간격
  - 스폰 확률 (0-100%)
  - 스폰 수량
  - 스폰 범위
- **웨이브 진행**: 제한 시간 동안 생존
- **웨이브 종료**: 모든 적 자동 처치 후 다음 단계로

### 4. 적 시스템
- **AI**: 플레이어를 추적하여 공격
- **스폰**: 플레이어 주변 일정 범위에서 스폰 인디케이터와 함께 출현
- **드랍**: 처치 시 경험치/골드/아이템 드랍

### 5. 드랍 시스템
- **자동 수집**: 플레이어 근처의 아이템은 자동으로 끌어당김
- **드랍 타입**: 
  - 경험치
  - 골드
  - 재료/아이템

## 📅 개발 로드맵

### Phase 1: 핵심 게임 루프 완성 (우선순위 높음)
1. **게임 상태 관리 구현**
   - GameState enum 정의 (MENU, GAME, SHOP, WEAPONSELECTION 등)
   - 상태 전환 로직 구현
   - 상태별 리스너 시스템 활성화

2. **상점 시스템 구현**
   - 웨이브 간 상점 UI
   - 무기 구매/판매
   - 아이템 구매
   - 골드 시스템 연동

3. **적 다양화**
   - 원거리 적 (공격 전 경고 표시)
   - 돌격형 적
   - 탱커형 적
   - 버프형 적
   - 회복형 적

4. **스폰 시스템 개선**
   - 스폰 한계 설정 및 관리
   - 적 무리 스폰 로직
   - 적 사망 시 추가 적 출현

### Phase 2: 콘텐츠 확장
1. **캐릭터 시스템**
   - 여러 플레이 가능 캐릭터 추가
   - 캐릭터별 고유 특성
   - 캐릭터 선택 UI

2. **무기 다양화**
   - 다양한 무기 타입 추가
   - 무기 레벨업 시스템
   - 무기 시너지 시스템

3. **아이템 시스템**
   - 패시브 아이템
   - 액티브 아이템
   - 아이템 조합 시스템

4. **보스 시스템**
   - 보스 적 추가
   - 보스 패턴 구현
   - 보스 보상

### Phase 3: 메타 진행 & 완성도
1. **메타 진행**
   - 영구 업그레이드 시스템
   - 잠금 해제 시스템
   - 업적 시스템

2. **UI/UX 개선**
   - 메인 메뉴
   - 결과 화면
   - 튜토리얼

3. **오디오**
   - 배경 음악
   - 효과음
   - 오디오 설정

4. **최적화**
   - 빌드 최적화 (Mono/IL2CPP)
   - 오브젝트 풀링
   - 성능 개선
